dbt Layers

Три слоя трансформации. Подробнее о стеке — в 01. Data Architecture.

Staging (stg_)

1:1 с источником. Очистка, типизация, дедупликация. Без бизнес-логики.

-- stg_betting__bets
SELECT
    bet_id,
    player_id,
    lower(product) AS product,
    CAST(bet_amount AS Decimal64(2)) AS bet_amount,
    CAST(payout_amount AS Decimal64(2)) AS payout_amount,
    lower(status) AS status,
    toDateTime(placed_at) AS placed_at,
    toDateTime(settled_at) AS settled_at
FROM raw.betting_bets
WHERE bet_id IS NOT NULL

Intermediate (int_)

Джойны, расчёты, обогащение данных.

-- int_bets_enriched
SELECT
    b.bet_id,
    b.player_id,
    p.geo,
    p.vip_tier,
    p.acquisition_channel,
    b.product,
    b.bet_amount,
    b.payout_amount,
    b.bet_amount - b.payout_amount AS ggr,
    b.is_bonus_bet,
    g.game_type,
    g.provider,
    b.placed_at
FROM stg_betting__bets b
LEFT JOIN stg_players p ON b.player_id = p.player_id
LEFT JOIN stg_games g ON b.game_id = g.game_id

Marts (mart_)

Финальные витрины для BI и DS. Каждый mart отвечает на конкретный бизнес-вопрос.

Mart-таблицы

▶ mart_daily_kpi daily refresh

Ежедневные бизнес-метрики. Основной дашборд CEO. Обновляется к 07:00 UTC.

Sources: int_bets_enriched, int_daily_costs

  • dt — дата
  • active_players — уникальные игроки за день
  • turnover — сумма ставок (handle)
  • ggr — Gross Gaming Revenue
  • ngr — Net Gaming Revenue (GGR - bonus - fees)
  • deposits, withdrawals — суммы
  • ftd_count — новые депозиторы
▶ mart_player_lifetime daily refresh

Lifetime-метрики по каждому игроку. Основа для LTV-моделей и сегментации.

Sources: int_bets_enriched, int_deposits_enriched, int_sessions_enriched

  • player_id — ID игрока
  • lifetime_turnover — все ставки за всё время
  • lifetime_ggr — cumulative GGR
  • lifetime_deposits — сумма депозитов
  • total_bets — количество ставок
  • days_active — дней с активностью
  • last_activity_at — последняя активность
  • churn_risk — score из ML-модели
▶ mart_cohort_analysis weekly refresh

Когортный анализ по дате регистрации. Retention, cumulative NGR, LTV по когортам.

Sources: int_bets_enriched, stg_players

  • cohort_month — месяц регистрации
  • period_number — месяц жизни когорты (0, 1, 2...)
  • cohort_size — размер когорты
  • retained_players — активные в период
  • retention_rate — % retained
  • cumulative_ngr — накопленный NGR
▶ mart_marketing_performance daily refresh

Эффективность маркетинга: CAC, LTV:CAC, ROAS по каналам и кампаниям.

Sources: int_deposits_enriched, stg_players, stg_marketing_spend

  • channel — канал привлечения
  • campaign — кампания
  • spend — маркетинговые расходы
  • registrations — регистрации
  • ftd_count — первые депозиты
  • cac — cost per acquisition
  • ltv_cac_ratio — LTV / CAC
  • roas — return on ad spend
▶ mart_game_performance daily refresh

Эффективность казино-игр: GGR по провайдерам, фактический vs теоретический RTP, топ-игры. Основной mart для продуктовой команды казино.

Sources: int_bets_enriched, stg_games

  • game_id — ID игры
  • provider — провайдер (Pragmatic Play, Evolution и др.)
  • game_type — slot / table / live / crash
  • total_bets — количество ставок
  • turnover — сумма ставок
  • ggr — GGR по игре
  • theoretical_rtp — теоретический RTP
  • actual_rtp — фактический RTP (payout / turnover)
  • unique_players — уникальные игроки
