Q4 — Scale

Масштабирование: real-time, Reverse ETL, fraud v2, оптимизация затрат. Подготовка фундамента на 2027.

▶ Real-time Analytics

Переход от batch (daily) к near-real-time для критичных метрик:

МетрикаСейчасЦельКак
NGRDaily (07:00)15 min lagKafka → ClickHouse materialized view
Active PlayersDailyReal-timeKafka Streams counter
Fraud Score<100ms (already)MaintainModel v2 with more features
Deposit monitoringHourly5 minClickHouse streaming insert
Real-time ≠ всё real-time. Только метрики, где latency напрямую влияет на решения (fraud, live ops). Для финансовой отчётности daily batch остаётся gold standard.
▶ Reverse ETL & Closed Loop

Замыкаем цикл: данные → модель → действие → результат → данные.

  • Churn → CRM: churn_score > 0.7 → CRM segment «At Risk» → персонализированный бонус
  • LTV → Ad Platforms: LTV segments → Facebook/Google Ads → bid optimization
  • Recommendations → Product: game recommendations → in-app placement → engagement uplift

Инструмент: Census (DWH → CRM, Ad platforms). Подробнее: Data Architecture (Reverse ETL).

▶ Cost Optimization (-20%)

Цель Q4: снизить infrastructure cost на 20% ($3.6K/мес от $18K/мес infrastructure, не от total budget) при росте данных 2x:

  • ClickHouse cold storage migration (TTL policies)
  • Reserved instances для стабильных workloads
  • ML training на spot instances
  • Quarterly query review — оптимизация top-20 heavy queries
  • Unused resource cleanup

Подробнее: FinOps.

▶ Data Mesh Evaluation

Evaluation (не migration) — готовность к data mesh при дальнейшем масштабировании:

  • Вопрос: при 50+ человек в data — нужна ли децентрализация?
  • Оценить: domain ownership maturity, self-serve platform readiness, team autonomy
  • Deliverable: Recommendation document для C-level (stay centralized / hybrid / start mesh transition)
Data mesh — не цель, а средство. При 20 людях централизованная модель работает отлично. Evaluation нужен, чтобы быть готовым к росту, а не чтобы модно выглядеть.
