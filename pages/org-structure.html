Стримы

▶ Data Engineering 5 человек

Команда

Lead Data Engineer
Архитектура DWH, код-ревью, планирование, on-call эскалации. Мост между DE и аналитикой — переводит бизнес-требования в технические задачи для инженеров.
Data Engineer × 2
Разработка и поддержка пайплайнов: Airbyte коннекторы, Kafka consumers, загрузка в ClickHouse. Мониторинг, on-call ротация.
Analytics Engineer
dbt-модели: staging → intermediate → marts. Тесты качества данных, документация моделей. Работает на стыке DE и аналитики.
Platform / DevOps Engineer
Инфраструктура data-платформы: Airflow, ClickHouse кластер, мониторинг (Grafana, alerting). CI/CD для dbt и пайплайнов.

Зона ответственности

  • Ingestion: все пайплайны из источников в DWH
  • DWH: схема, производительность, партиционирование ClickHouse
  • Инфраструктура: Airflow, Kafka, мониторинг
  • DQ на уровне пайплайнов: свежесть, полнота, схема
  • On-call: ротация 1 неделя, реагирование на P1/P2 инциденты

Взаимодействие

  • Backend: доступ к данным, API, схемы таблиц
  • DevOps: инфраструктура, деплой, сети
  • Analytics: требования к mart-таблицам, SLA по свежести

Ключевые deliverables

  • Пайплайны работают 24/7
  • Данные в DWH свежие к 07:00 UTC
  • SLA uptime: 99.5%
▶ Analytics 8 человек

Команда

Lead Analyst
Приоритизация задач аналитиков, методология исследований, ревью отчётов. Представляет данные на C-level. Стратегические инсайты.
Product Analyst × 3
По направлениям: Sports, Casino, Payments. Метрики продукта, воронки, когортный анализ, A/B-тесты. Работают в связке с продуктовыми менеджерами.
Marketing Analyst × 2
Acquisition: CAC, ROAS, атрибуция по каналам. Retention/CRM: churn-анализ, бонусная эффективность, LTV по когортам.
Financial Analyst
P&L, unit economics, бюджетирование. NGR breakdown по продуктам и рынкам. Финансовая отчётность для CFO.
BI Developer
Дашборды в Metabase, семантический слой. Стандартизация отчётов, шаблоны. Мост между аналитиками и BI-инженерами.

Зона ответственности

  • Регулярная отчётность: daily/weekly/monthly
  • Исследования: deep dives, impact analysis
  • Ad-hoc: ответы на вопросы бизнеса (SLA: 3 рабочих дня)
  • Когортный анализ, воронки, сегментация

Взаимодействие

  • Product: метрики фичей, A/B-тесты, user research
  • Marketing: эффективность каналов, CRM-кампании
  • Finance: P&L, unit economics, прогнозы
  • C-level: стратегические отчёты, OKR-трекинг

Ключевые deliverables

  • Дашборды актуальны и доступны
  • Weekly/monthly отчёты вовремя
  • Ad-hoc: ≤ 3 рабочих дня
  • Исследование: ≤ 2 недели
▶ Data Science 4 человека

Команда

Lead Data Scientist
Методология, приоритизация ML-задач, ревью моделей. Координация с Product по use cases. Определяет, когда ML оправдан, а когда хватит эвристики.
ML Engineer
Деплой моделей в продакшн, MLOps инфраструктура: MLflow, Feature Store, мониторинг дрифта. Мост между DS-экспериментами и production.
Data Scientist — LTV & Churn
Предиктивные модели: LTV prediction, churn scoring, recommendation engine. Работает с product-аналитиками на когортных данных.
Data Scientist — Fraud & Risk
Real-time fraud scoring, anomaly detection, risk models. Latency SLA < 100ms. Работает с Risk/Compliance командой.

Зона ответственности

  • Предиктивные модели: LTV, churn, fraud, recommendations
  • A/B тестирование: методология, расчёт sample size, анализ
  • Experimentation platform

Взаимодействие

  • Product: use cases, приоритизация моделей
  • DE: данные, Feature Store, пайплайны для моделей
  • Risk/Compliance: fraud detection, ответственный гейминг

Ключевые deliverables

  • Модели в production
  • A/B отчёты с рекомендациями
  • Fraud scoring: latency < 100ms, FPR < 5%
▶ BI Engineering 3 человека

Команда

Lead BI Engineer
Архитектура BI-слоя, стандарты визуализации, семантический слой. Координация с Lead Analyst по метрикам.
BI Developer
Сложные дашборды в Metabase, embedded analytics, кастомные визуализации. Оптимизация запросов для быстрой загрузки.
Data Literacy Specialist
Обучение бизнес-пользователей self-service аналитике. Воркшопы, документация, шаблоны. Цель — 80% ad-hoc через self-service.

Зона ответственности

  • Семантический слой: единые определения метрик
  • Стандартизация отчётности: шаблоны, гайдлайны
  • Self-service инструменты: Metabase для бизнес-пользователей
  • Data literacy программа: обучение, воркшопы

Взаимодействие

  • Все бизнес-функции: конечные пользователи дашбордов
  • Analytics: требования к метрикам, валидация
  • DE: производительность запросов, SLA по обновлению

Ключевые deliverables

  • Self-service покрытие: 80% ad-hoc через Metabase
  • Data literacy: 3 воркшопа в квартал
  • Стандартизированные шаблоны отчётов

Career Ladder

Единая карьерная лестница для всех стримов. Каждый уровень — это не только про hard skills, но и про scope of impact.

Junior
  • Выполняет задачи под руководством ментора
  • Учится стандартам и процессам команды
  • Scope: отдельные задачи (запрос, модель, тест)
  • Срок на уровне: 1-2 года
Middle
  • Самостоятельно решает задачи без постоянного ревью
  • Понимает бизнес-контекст своих задач
  • Scope: фича / исследование / pipeline целиком
  • Менторит джуниоров, участвует в код-ревью
  • Срок на уровне: 2-3 года
Senior
  • Определяет техническое решение для сложных задач
  • Влияет на архитектуру и процессы команды
  • Scope: проект / направление (все спорт-аналитика, вся ingestion)
  • Менторит middle, проводит техревью, пишет RFC
  • Может работать кросс-функционально
Lead
  • Отвечает за стрим: планирование, приоритизация, people management
  • Scope: весь стрим (DE, Analytics, DS, BI)
  • Нанимает, проводит 1-on-1, растит людей
  • Представляет стрим перед Director и стейкхолдерами
  • 60% management / 40% hands-on (зависит от зрелости команды)
Head / Director
  • Стратегия data-функции, OKR, бюджет
  • Scope: вся data-организация (20 человек, 4 стрима)
  • C-level коммуникация, приоритизация на уровне компании
  • Найм лидов, организационный дизайн
  • 90% management / 10% hands-on (архитектурные решения)
